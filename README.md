# 🎲 요트 다이스 (Yacht Dice)

3D 물리감 주사위로 즐기는 요트 다이스. **로컬 / AI 대전 / 온라인 멀티플레이**를 모두 지원합니다.

## 기능
- **실감나는 3D 주사위** — three.js로 만든 입체 주사위가 던지면 튀어오르며 굴러서 멈춥니다. 굴릴 때 달그락거리는 소리도 같이 납니다.
- **주사위 고정** — 주사위를 탭(클릭)하면 고정되고, 남은 주사위만 다시 굴립니다. (한 턴 최대 3회)
- **15초 제한시간** — 차례마다 15초 타이머. 시간이 끝나면 자동으로 굴리고 점수까지 기록합니다.
- **AI 상대** — 사람 대신 AI를 넣을 수 있습니다. AI는 확률을 계산해 남길 주사위와 점수 칸을 스스로 선택합니다.
- **규칙 모드 3종**
  - **요트 다이스** — 국내 앱 규칙(포카드·풀하우스 합산, 상단 63점↑ 보너스 +35)
  - **야찌(Yahtzee)** — 해즈브로 규칙(트리플 추가, 풀하우스 25·스트레이트 30/40 고정)
  - **오리지널** — 1938 원조 규칙(포카드 4개 합, 스트레이트 정확히 30, 보너스 없음)
- **온라인 멀티플레이** — 방 코드를 공유해 친구들과 같은 게임을 실시간으로(최대 6명).
- **창 투명도 조절** — 카카오톡 PC창처럼 우측 상단 슬라이더로 전체 투명도를 조절합니다.

## 빠르게 실행하기

필요한 것: [Node.js](https://nodejs.org) 18 이상.

```bash
npm install
npm start
```

브라우저에서 **http://localhost:3000** 접속.

- **로컬 / AI**: 바로 플레이. 사람·AI를 섞어서 2~6명.
- **온라인**: `방 만들기`로 코드를 받아 친구에게 공유 → 친구는 `방 참가`에 코드 입력. 호스트가 규칙을 고르고 `게임 시작`.

> 같은 와이파이 안에서 테스트하려면, 호스트 PC의 로컬 IP(예: `http://192.168.0.5:3000`)로 다른 기기에서 접속하세요.

## 인터넷에 배포하기 (친구들과 진짜로 같이 하기)

이 프로젝트는 정적 파일과 WebSocket을 한 서버(`server.js`)에서 같이 제공합니다. Node + WebSocket을 지원하는 곳이면 어디든 올릴 수 있습니다.

### Render (무료, 추천)
1. 이 폴더를 GitHub 저장소로 올립니다.
2. [render.com](https://render.com) → New → **Web Service** → 저장소 연결.
3. 설정:
   - Build Command: `npm install`
   - Start Command: `node server.js`
4. 배포되면 나오는 주소(`https://....onrender.com`)를 친구에게 공유하면 끝.

### Railway / Fly.io / 기타
- Start Command를 `node server.js`로 지정하면 됩니다.
- 포트는 환경변수 `PORT`를 자동으로 읽습니다.

### 로컬 PC를 잠깐 외부 공개 (배포 없이)
```bash
npm start          # 터미널 1
npx localtunnel --port 3000   # 터미널 2 → 나오는 https 주소 공유
```

## 조작법
- **굴리기 버튼**: 주사위 던지기 (턴당 3회)
- **주사위 탭/클릭**: 고정 / 해제
- **점수표의 초록 숫자 탭**: 그 칸에 점수 기록 → 다음 사람
- **우측 상단 🪟 슬라이더**: 창 투명도

## 구조
```
yacht-dice/
├─ public/
│  └─ index.html   # 게임 전체 (three.js는 CDN에서 로드)
├─ server.js       # 정적 서버 + WebSocket 릴레이(방 관리)
├─ package.json
└─ README.md
```

## 참고
- 3D 라이브러리(three.js)는 CDN에서 불러오므로 **인터넷 연결**이 필요합니다.
- 온라인 모드는 서버를 통해서만 동작합니다. `index.html`을 파일로 직접 열면(file://) 로컬/AI 모드만 됩니다.
- 온라인 동기화는 호스트가 게임 진행을 맡는 방식(host-authoritative)입니다. 호스트가 나가면 다음 사람에게 호스트가 넘어갑니다.
