# 알까기 (Alkkagi) 설계서

> 주사위 골목 6번째 게임. 5종(윷·요트·너클본즈·라이어·좌중우)과 같은 아키텍처(단일 HTML + `*-core.js` + 서버 권위형)를 따르되, **연속 물리 시뮬레이션**이라는 결정적 차이 하나가 있다.

## 확정된 결정

- **물리**: 서버 결정론 시뮬(방식 B) + 5종과 동일한 **엔진 이중구조**. 클라·서버 양쪽에 `simulate()` 순수 함수를 두고, 최종 결과는 서버 스트림을 신뢰. 클라 로컬 시뮬은 즉시 프리뷰용.
- **렌더**: Canvas 2D. (SVG는 20돌 × 60Hz 애니에 무겁다)
- **1단계 착수**: 엔진 뼈대 + 로컬 vs 랜덤 AI만. 온라인·아이템·오늘의 규칙은 이후 단계.

## 게임 규칙

- 2인. 각자 자기 편 돌을 손끝으로 튕겨 상대 돌을 판 밖으로 밀어낸다.
- 판 밖으로 나간 돌은 그 판에서 탈락.
- **상대 돌이 다 떨어지면 승**. 내 돌이 다 떨어지면 패.
- **벽 반사 없음** (전통 규칙 — 판 가장자리에서 그대로 낙사).


## 물리 사양

**좌표**: 판 (0, 0) ~ (100, 100) 단위. 표준 돌 반경 3.5.

**타임스텝**: 고정 60Hz(dt = 1/60초). 최대 180프레임(3초). 종료 조건은 모든 돌 속력 < `STOP_EPS`(0.5) 또는 3초 도달.

**마찰**: 매 스텝 `v *= FRICTION_FACTOR`(0.98). 1초 후 약 30%, 3초 후 2.6% 남음.

**충돌** (원-원 탄성):
```
dist = |B.pos - A.pos|
if dist < A.r + B.r:
  n = (B.pos - A.pos) / dist
  rvn = (B.v - A.v) · n
  if rvn < 0:                    # 접근 중일 때만
    j = -(1 + E) * rvn / (1/A.m + 1/B.m)
    A.v -= j/A.m * n
    B.v += j/B.m * n
  # 겹침 즉시 분리
  overlap = A.r + B.r - dist
  A.pos -= n * overlap * (B.m/(A.m+B.m))
  B.pos += n * overlap * (A.m/(A.m+B.m))
```
탄성 계수 `E`= 0.9.

**낙사**: 매 스텝 끝에 돌 중심이 판 밖이면 `alive=false`, 이후 시뮬에서 제거. `events`에 `{t,type:'off',id}` 기록.

**플릭 상한**: 파워 0..1 → 초기 속도 0..80 u/s. 최소 파워 0.05 미만은 무시.

**결정론**: 순수 함수(입력 상태 + 플릭 벡터만으로 결과 확정). `Math.random` 절대 금지 — 오늘의 규칙 랜덤은 상위에서 시드 넣고 결정론적 PRNG 사용.

## 시뮬 결과 페이로드

```js
simulate(state, flick) → {
  frames: [[{id,x,y,vx,vy}...], ...],   // 180개
  events: [{t,type,id,extra?}, ...],    // off / hit / stopped
  final:  [{id,x,y,alive}, ...],
  next:   { phase, turn, ... }          // 시뮬 이후의 새 상태 힌트
}
```
표준 판(6+6=12돌, 3초, 60Hz) = **약 35KB/턴**. 압축 여지: 정지 돌 첫 프레임만 · 좌표 int8 델타 인코딩.

## 온라인 프로토콜

```
클라 → 서버 : {t:'action', a:{type:'flick', stoneId, angle, power}}
서버 → 클라 : {t:'state', s:...}                          # 상태 스냅샷
              {t:'sim',   frames, events, final, next}    # 결과 스트림
```

- 서버는 `flick` 유효성만 검증(자기 돌인지·전진 각도인지·파워 범위)하고 `simulate()` 실행 후 브로드캐스트.
- 시뮬 3.5초 안에 응답 없으면 강제 종료.
- 재접속: 진행 중 sim은 프레임 재전송 or 결과부터 skip.

## 클라이언트 UX

- **선택 → 조준 → 발사**: mousedown → 드래그(고무줄식, 반대 방향 화살표) → mouseup.
- **파워**: 드래그 거리로. 상한/게이지 표시.
- **취소**: 돌 위 벗어나면 자동 취소.
- **가이드라인**(옵션): 다음 충돌까지 점선 프리뷰. `alk_guide=1` 기본 켬.
- **모바일**: `touch-action:none` 필수 (판 스크롤 방지).
- **모션 절제**: `prefers-reduced-motion`에선 낙사 애니 생략, 즉시 사라짐.

## 검증

- **결정론 단언**: 고정 입력 × 100회 → 최종 좌표 완전 일치.
- **대칭성**: 같은 실력 랜덤 봇 100판 → 승률 50±5%.
- **무한 루프 없음**: 모든 판 3초 안 종료.
- **온라인 e2e**: `verify-online.js`에 시나리오 추가.
- **연출**: `capture.js`로 얼음 궤적·폭탄 캡처. Reduced-motion 가드 확인(CLAUDE.md #7).
- **드리프트**: `check:drift`에 alkkagi 포함.

## 리스크

1. **결정론 부동소수** — CPU/브라우저별 마지막 자리 흔들림. 실용안: 서버 결과 신뢰, 클라 로컬 시뮬은 프리뷰용.
2. **성능** — O(n²) 충돌 × 180스텝. 방 하나당 저부하이지만 여러 방 동시엔 이벤트 루프 청크 분할(`setImmediate`).
3. **모바일 스크롤** — `touch-action:none` 필수.
4. **연출 헤드리스** — `prefers-reduced-motion:reduce` 기본. CLAUDE.md #7 재확인.
5. **엔진 이중구조** — 5종 함정 그대로. `*-core.js`만 고치고 `npm run build`.
