## 0. 이 문서가 존재하는 이유

주사위 골목은 22일 넘게 라이브였지만 **방문자 수를 한 번도 모른 적이 없다.**
게임이 재미없어서 안 되는 건지, 아무도 온 적이 없어서 모르는 건지 구분이 안 된다.
이 실험의 산출물은 "성공/실패"가 아니라 **숫자 그 자체**다.

설계 문서의 표본 하한: **총 유입 100명 미만이면 판정 유보.**
그때 문제는 게임이 아니라 유입 방법이며, 게임에 대한 결론을 내리지 않는다.

## 1. 계측된 퍼널 (2026-07-22 구현 완료)

| 단계 | 이벤트 | 어디서 |
|---|---|---|

> ⚠️ `{게임}` 같은 커스텀 속성은 Plausible 요금제에 따라 대시보드에서 안 보일 수 있다.

> 속성은 보너스이지 이 실험의 필수 조건이 아니다.

## 2. 실험 1건의 정의 (설계 문서 인용)

> **1건 = 고유 UTM을 가진 단일 채널 게시물, 노출 확인 가능한 것**

## 3. 실험 3건

각 실험은 **가설 → 채널 → 착지 → UTM → 노출 확인법**을 미리 적고 시작한다.
사후에 해석을 바꾸지 않기 위해서다.

### 실험 A — "회사에서 몰래" (위장 모드)

가장 차별적인 자산이고, 아직 한 번도 그 앵글로 알려본 적이 없다.

| | |
|---|---|
| **가설** | 사무실에서 짧게 놀 거리를 찾는 직장인은 '위장되는' 게임이면 클릭한다 |
| **채널** | 직장인 커뮤니티 (블라인드 / 클리앙 모두의공원 / 디시 회사원 갤) — 1곳만 |
| **착지** | **`/yacht.html` 직행** (허브 아님 ← 아래 주의) |
| **UTM** | `?utm_source=blind&utm_medium=post&utm_campaign=disguise` |
| **노출 확인** | 게시물 조회수 스크린샷 |
| **핵심 지표** | 진입 → `게임시작` 전환율. 위장 앵글이 실제 플레이로 이어지는가 |

> ⚠️ **허브로 보내지 말 것.** 위장 모드(엑셀·포토샵)는 `yacht.html`에만 있다.
> 허브는 대놓고 게임처럼 생겼다. "몰래 하는 게임"을 약속하고 허브에 떨구면
> 첫 화면에서 약속이 깨진다. 허브 요트 카드에 힌트 pill은 달아뒀지만
> 이 실험의 착지는 요트 직행이다.

### 실험 B — 윷놀이 (계절재 아닌 시기의 온라인 대전)

가장 완성도가 높고(v1.16x, 대전 방식 3택) 한국인 검색 수요가 명확하다.

| | |
|---|---|
| **가설** | 설치 없이 링크로 부르는 온라인 윷놀이 수요는 명절이 아니어도 존재한다 |
| **채널** | 보드게임/커뮤니티 (루리웹 유머·게임게시판 중 1곳) |
| **착지** | `/yut.html` |
| **UTM** | `?utm_source=ruliweb&utm_medium=post&utm_campaign=yut-online` |
| **노출 확인** | 조회수 + 댓글 수 |
| **핵심 지표** | `1판완료` 도달률, D1 재방문 |

### 실험 C — 허브 (다섯 게임 묶음 소구)

앞 둘이 단일 게임 소구라면, 이건 "골목" 자체가 팔리는지 본다.

| | |
|---|---|
| **가설** | 게임 5종 묶음이라는 사실 자체가 클릭 이유가 된다 |
| **채널** | 숏폼 1건 (인스타 릴스 또는 틱톡) — 주사위 굴리는 화면 15초 |
| **착지** | `/` (허브) |
| **UTM** | `?utm_source=reels&utm_medium=video&utm_campaign=hub5` |
| **노출 확인** | 릴스 조회수 |
| **핵심 지표** | `허브_카드클릭` 분포 — **어느 게임이 실제로 선택되는가** |

> 실험 C의 부산물이 제일 값질 수 있다. 5종 중 뭘 남기고 뭘 버릴지에 대한
> 최초의 외부 데이터가 나온다.

## 4. 순서와 간격

**동시에 쏘지 않는다.** 겹치면 어느 채널이 먹혔는지 UTM으로도 분리가 흐려지고,
무엇보다 첫 실험에서 배운 걸 다음 실험에 못 쓴다.

| 주차 | 할 일 |
|---|---|
| 1주차 | 계측 배포 → **48시간 그냥 둔다** (기존 유입 베이스라인 확보) |
| | 실험 A 발사 |
| 2주차 | A 판독 → 실험 B 발사 |
| 3주차 | B 판독 → 실험 C 발사 |
| 4주차 | C 판독 · 누적 집계 |
| 08-17 이후 | 설계 문서의 **B 재개 논증 요건** 3개를 쓸 수 있는지 판단 |

## 5. 발사 전 체크리스트

- [ ] Plausible 계정 생성 · 사이트 등록 (`yacht-dice-jxva.onrender.com`)
- [ ] 배포 후 실제 방문 1회로 대시보드에 찍히는지 눈으로 확인
- [ ] **콜드스타트 실측** — 24시간 무접속 후 첫 로딩 시간 1회 측정
      (Render 무료 티어는 15분 놀면 잠든다. 커뮤니티 유입은 대부분 콜드로 떨어진다.
      첫 로딩이 느리면 이 실험은 게임이 아니라 서버를 측정하게 된다)
- [ ] 처리방침 페이지 접근 확인 (`/privacy.html`)
- [ ] 각 커뮤니티의 홍보 금지 규정 확인 — 계정 정지되면 채널을 잃는다

## 7. 이 실험이 답하지 않는 것

- **B2B 광고게임 수요** — 그건 C 트랙(전화 한 통)의 몫이다. 이 실험과 무관하다.
- **돈을 낼 의향** — 결제 기능이 없으므로 측정 불가.
- **게임의 재미** — 유입이 100명 미만이면 아무것도 말하지 않는다.
