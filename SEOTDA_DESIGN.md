## 패 랭킹 (높은 순)

패평가 `evalHand(a, b) → { tier, sub, name }`. tier가 클수록 강하고, 같은 tier면 sub로 비교.

1. **광땡** (광 2장) — `38광땡(3,8) > 18광땡(1,8) > 13광땡(1,3)`
2. **땡** (같은 월 페어) — `장땡(10) > 9땡 > … > 삥땡(1)`
3. **특수 끗** (아래 조합, 순서대로) — `알리(1,2) > 독사(1,4) > 구삥(1,9) > 장삥(1,10) > 장사(4,10) > 세륙(4,6)`
4. **끗** (특수 아님) — `(m1+m2)%10`. `갑오(9) > 8끗 > … > 1끗 > 망통(0)`

### 잡이 (토글 · 가위바위보식 반전)

> 출처(secret-tip.tistory.com/5, ansqo080808.tistory.com/17) 대조로 확정. 네이버 블로그는 차단되어 미반영.

쇼다운에서 **base 랭킹으로 1차 정렬 후, 잡이 오버라이드를 쌍대(pairwise)로 적용**한다.

피망은 **전부 월 기반**(`섯다_특수족보` 스샷: 땡잡이 3월+7월 / 암행어사 4월+7월 / 구사 4월+9월).

- **암행어사** `{4,7}` — **13·18광땡을 잡음**. **38광땡엔 짐.** 광땡이 판에 없으면 최악패(합11=**1끗**). *(주의: tarkerx 소스만 "13/18광땡 못 잡음"이라 반대로 적혔으나, 나머지 3개 소스+피망 인게임은 "잡음" → 표준=잡음 채택.)*
- **땡잡이** `{3광, 7}` — **구땡(9땡) 이하의 땡만 잡음**. **장땡(10땡)·광땡·암행어사엔 짐.** 땡이 없으면 **망통(합0)**. *(카드: 3은 **3광**이어야 함 — 텍스트 룰서 다수. 피망 인게임 라벨은 '3월'로도. 우린 광 추적하니 3광+7로 정확히.)*
- **구사** `{4,9}` — 판 **최고패가 '알리 이하'** 면 그 판 **재경기**(팟 유지, 앤티만 다시). **땡·광땡이 판에 있으면 발동 안 함**(그냥 짐).
- **멍텅구리 구사** — **4열끗 + 9열끗**(특정 열끗 카드). 최고패 **'구땡(9땡) 이하'** 면 재경기(구사보다 강함). → 카드 모델에 `type` 필요. **1단계엔 일반 구사만**, 멍텅구리는 `type` 추가 후 4단계.

> **카드 모델**: 기본 `{월, 광}` 이면 암행어사·땡잡이·(일반)구사 전부 월 기반이라 표현 가능. **멍텅구리 구사만 열끗 구분** 필요 → 후에 `type:'광'|'열끗'|'띠'|'피'` 추가.

> 잡이는 **직관을 깨는** 룰이라 기본 꺼둠. 켜면 셋업/툴팁에 "무엇이 무엇을 잡는지" 표로 안내.

## 베팅 (피망 2장 섯다 · 원본 대조 확정)

> 피망 스샷(`섯다_규칙1`, `섯다_플레이.mp4`) 기준. **베팅은 2라운드** — 1장 받고 첫 베팅, 2장째 받고 둘째 베팅.

**흐름 (한 판)**
1. **앤티**: 각자 기본 판돈 `ante` 냄 → 초기 팟.
2. **1장 딜** → **첫 베팅**.
3. **2장째 딜** → **둘째 베팅**.
4. **쇼다운**: 안 죽은 사람 패 공개 → 최고 패가 팟 획득. **1명 빼고 다 죽으면 즉시 승**(패 안 깜).

**베팅 규칙 (원본)**
- **첫 베팅**: 최초(선)는 **하프만** 가능(또는 다이). 이후 참가자는 **하프 / 콜 / 다이**.
- **둘째 베팅부터**: 전체 참가자 자유 — **다이 · 삥 · 따당 · 콜 · 쿼터 · 하프**.
  - (일부 방은 첫 베팅도 '콜·다이'만 제공 → 옵션.)

**액션 (하단 베팅 바 6버튼, 피망 순서)**
| 버튼 | 뜻 |
|---|---|
| **다이** | 폴드(포기, 낸 칩 회수 X) |
| **삥** | 최소 베팅(= ante 크기) |
| **따당** | 직전 베팅의 **2배** 레이즈 |
| **콜** | 현재 베팅액 맞춤 |
| **쿼터** | **팟의 1/4** 레이즈 |
| **하프** | **팟의 1/2** 레이즈 |
| (체크) | 낼 게 없을 때 넘김(내 기여 == 현재 베팅) |
| (올인) | 칩 부족 시 남은 전부 |

**땡값 (옵션)**: **땡 이상**으로 승리 시, 끝까지 따라온 패자들이 **땡값(추가 징수)** 지불. 단 — 패자가 **올인**이면 안 받음, **1땡~9땡끼리 승부**면 안 받음(예: 승자 9땡·패자 1땡). 합의로 끄기 가능 → **토글**.

**베팅 상태머신** — `{ pot, bet(현재 최대 기여액), turn, round(1|2), contrib[seat], folded[seat], allin[seat] }`. 라운드 종료: 살아있는 전원 `contrib == bet`(또는 올인) & 한 바퀴 돎.

**올인/사이드팟**: 1단계엔 **단순화**(올인 초과 콜은 메인팟, 부족분은 그 사람 몫만큼만 경쟁). 정식 사이드팟은 4단계.

## 히든 정보 / 직렬화 (핵심)

`serialize(viewer)` — **`ld` 패턴을 그대로**:
- `me`: viewer의 2장 **앞면**(month/gwang 노출).
- `others`: 카드 **뒷면**(장수만, month/gwang 비노출). **쇼다운·공개 전까지 절대 안 보냄.**
- 공통: `pot, bet, turn, phase(ante|bet|showdown|over), players[{pid,name,chips,contrib,folded,allin}], lastAction, dealerSeat`.
- **쇼다운 시**: 안 죽은 전원 카드 + 패 이름(`장땡!` 등) 공개. 죽은 사람은 안 깜(선택 공개 옵션).
- 서버: `ld`처럼 **멤버마다 `serialize(mm.pid)` 로 개별 전송**. 로컬: 단일 엔진이 render 때 자기 것만 앞면, AI는 뒷면.

## 결정론

- **셔플**: Fisher-Yates + **주입 rng**(`opt.rng`, 서버가 시드 공급). `Math.random` 금지(5종 규약).
- 히든이라 클라가 딜을 검증 못 함 → **서버 권위**(다른 게임과 동일 신뢰 모델). 로컬은 단일 엔진.

## AI

- 자기 2장 → **패 강도**(190조합 중 백분위 0..1) 계산.
- 정책(난이도로 조절):
  - **강패(상위~25%)**: 따당/하프 레이즈, 가끔 슬로우플레이(콜).
  - **중간**: 콜, 큰 레이즈엔 폴드 확률.
  - **약패(하위~40%)**: 레이즈엔 다이. **블러프 확률 p**로 약패도 레이즈.
  - easy=블러프 적고 잘 죽음 / hard=블러프·압박 강함, 팟오즈 고려.
- **결정론**: AI 랜덤도 시드 rng 사용(재현/온라인 일관).

## 카드 아트 (CSS 간이 화투)

- **앞면**: 둥근 카드, 월별 배경색(전통 화투 톤: 1월 적송·2월 홍매·3월 벚꽃…), 월 숫자 크게(한자 또는 아라비아), 상단 작은 모티프(SVG/이모지), **광이면 금테 + `光`**, 피/열끗 작은 마커.
- **뒷면**: 통일된 화투 백(붉은 바탕 + 문양). 상대·미공개 카드용.
- **딜/뒤집기 애니**: 카드가 중앙에서 각자에게 날아가고(딜), 쇼다운 때 뒤집기(flip).
- 랭킹은 **광 여부만** 보므로 나머지 그림은 순수 장식 → 아트 부담 최소.

## 클라이언트 UX / 연출

> 피망 실제 화면(`섯다_플레이.mp4`, `섯다_쪼기.mp4`) 참고. 우리는 CSS 간이 화투 + 허브 캐릭터로.

- **테이블**: 초록 펠트 오벌. 플레이어를 둘레에 배치(**내가 하단 중앙**). 각자 **아바타(원형)+이름+칩**. 카드 2장(내 것만 앞면, 남은 **빨간 뒷면**).
- **중앙**: **판돈(팟)** 큰 숫자 + **콜비용**. 선(先) 마커를 현재 딜러/차례에.
- **내 패 이름 표시**: 내 카드 아래 `장사(10·4)` 처럼 **완성 족보명**을 실시간 표기(피망 동일).
- **족보/승률 패널**(우측, 옵션): 족보 목록 + 내 승률(계산). 초보 도움 → 토글.
- **베팅 바(하단 6버튼, 피망 순서)**: `다이 · 삥 · 따당 · 콜 · 쿼터 · 하프` + 각 버튼에 금액 표시. 유효하지 않은 액션은 비활성.
- **쪼기(핵심 손맛)**: 내 카드를 **손끝으로 천천히 밀어 숫자를 까보는** 연출. 모바일 드래그로 카드 모서리부터 점점 공개 → 섯다 특유의 긴장. `capture.js`로 검증.
- **딜 애니**: 중앙 덱 → 각자에게 카드 날아감(1장 → 첫 베팅 → 2장째).
- **쇼다운**: 안 죽은 사람 카드 **flip** → 패 이름 배너(`38광땡!` `장땡!` `알리!`), 승자에게 **칩(팟)이 흘러가는** 연출. 땡값 켜면 패자→승자 추가 이동.
- **다이**: 카드 엎고 흐리게. **잡이 반전**: `암행어사가 광땡을 잡았다!` 같은 특별 배너.
- **모션 절제**: `prefers-reduced-motion`이면 flip/쪼기/이동 생략, 즉시 표시.

## 검증

- **패평가 전수 테스트**: 20C2 = **190조합 전부** 랭킹 계산 → (a) 전순서 일관(비교 반사·추이성), (b) 알려진 관계 단언(38광땡 최강, 망통 최약, 장땡>9땡, 알리>세륙, 갑오=9끗 등).
- **잡이 토글**: on일 때 암행어사→13/18광땡 승, 땡잡이→땡 승, 구사→무효; off일 때 base 랭킹만.
- **결정론 셔플**: 시드 고정 → 동일 딜 재현.
- **베팅 SM**: 콜/따당/하프/다이 상태 전이, 팟 정합(합=Σ기여), 1명 남으면 즉시 종료, 라운드 종료 판정.
- **히든**: `serialize(A)`에 B의 month/gwang 안 담김(쇼다운 전). 쇼다운 후 공개.
- **온라인 e2e**(`verify-online.js`): 딜→베팅→쇼다운, 재접속 자기 패 복원, 관전자 시점.
- **AI**: 강패 승률↑, 무한 베팅/파산 루프 없음, 시드 재현.
- **드리프트**: `check:drift`에 seotda 포함(7종 OK).

## 로드맵

**1단계** (엔진) ← *다음*
1. `seotda-core.js` — 덱·셔플(시드)·`evalHand`·베팅 SM·기본 AI.
2. **패평가 190조합 테스트** + 결정론 셔플 테스트(헤드리스).

**2단계** (로컬 UX)
3. `seotda.html` — 카드 렌더(CSS 간이 화투)·베팅 UI·쇼다운. 로컬 vs AI만.
4. 딜/뒤집기/승리 연출. `build-inline`·`check-drift` 등록.

**3단계** (온라인)
5. `server.js` 통합(ld 패턴, per-member). `verify-online.js` 시나리오. 재접속·타임아웃·부정 방지(자기 턴·유효 액션만).

**4단계** (완성도)
6. 잡이 토글 UI + 안내표, 칩/레벨(메타), 사이드팟, 결과 히어로·이모지·캐릭터 캐스팅.

## 리스크

1. **히든 정보** — `ld` 선례로 리스크 낮음. `serialize(viewer)`에서 남의 카드 절대 누출 금지(테스트로 못박기).
2. **도박성** — 가상 칩·무료. 실제 재화 없음. 셋업/결과 문구 명시.
3. **카드 아트** — CSS 간이로 대체(광만 랭킹 영향). 월별 색/모티프는 장식.
4. **베팅 SM 복잡도** — 상태 전이·팟 정합을 단위 테스트로. 올인/사이드팟은 4단계로 미룸(초기 단순화).
5. **잡이 룰 혼란** — 기본 끔 + 토글 + 안내표. 켠 판만 반전 적용.
6. **엔진 이중구조** — 5종 함정 그대로. `*-core.js`만 고치고 `npm run build`.
